# Tests

## Overview
This directory contains all testing-related code for the Lost and Found Management System.

---

## Purpose
- Ensure system reliability and correctness
- Validate functional and non-functional requirements
- Detect bugs early

---

## Types of Tests
- Unit Tests
- Integration Tests
- API Tests
- (Optional) End-to-End Tests

---

## Scope
- Backend API testing
- Business logic validation
- Frontend component testing (if applicable)

---

## Tools (Suggested)
- JUnit / Spring Boot Test (Backend)
- Postman / REST Clients (API testing)
- Jest / React Testing Library (Frontend)

---

## Notes
- Aim for high test coverage (≥80% recommended)
- Tests should align with SRS requirements
